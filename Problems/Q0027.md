# 27. Remove Element (要素の削除)

**難易度:** Easy

## 問題文

整数配列 `nums` と整数 `val` が与えられます。`nums` 内の `val` と等しいすべての要素を**インプレース**（追加のメモリを使用せず、元の配列を直接変更すること）で削除してください。要素の順番は変更されても構いません。その後、`nums` 内の `val` と等しくない要素の数を返してください。

`nums` 内の `val` と等しくない要素の数を `k` とします。解答が承認されるには、以下の条件を満たす必要があります。

* 配列 `nums` を変更し、`nums` の最初の `k` 個の要素が `val` と等しくない要素になるようにします。`nums` の残りの要素（`k` 個より後ろの要素）や配列のサイズは重要ではありません。
* `k` を返します。

### カスタムジャッジ（判定基準）

ジャッジシステムは、以下のコードを用いてあなたのソリューションをテストします。

```java
int[] nums = [...]; // 入力配列
int val = ...; // 削除する値
int[] expectedNums = [...]; // 正しい長さを持つ期待される答え。
                            // val と等しい値を持たず、ソートされています。

int k = removeElement(nums, val); // あなたの実装を呼び出します

assert k == expectedNums.length;
sort(nums, 0, k); // nums の最初の k 個の要素をソートします
for (int i = 0; i < actualLength; i++) {
    assert nums[i] == expectedNums[i];
}
```

## 例

### 例 1:

* **入力:** `nums = [3,2,2,3]`, `val = 3`
* **出力:** `2`, `nums = [2,2,_,_]`
* **説明:** 関数は `k = 2` を返し、`nums` の最初の2つの要素は `2` になるべきです。返された `k` より後ろの要素が何であるかは重要ではありません（そのためアンダースコアで表されています）。

### 例 2:

* **入力:** `nums = [0,1,2,2,3,0,4,2]`, `val = 2`
* **出力:** `5`, `nums = [0,1,4,0,3,_,_,_]`
* **説明:** 関数は `k = 5` を返し、`nums` の最初の5つの要素は `0`, `0`, `1`, `3`, `4` を含んでいるべきです。これら5つの要素は**どのような順序で返されても構いません**。返された `k` より後ろの要素が何であるかは重要ではありません（そのためアンダースコアで表されています）。

---

## 制約

* `0 <= nums.length <= 100`
* `0 <= nums[i] <= 50`
* `0 <= val <= 100`
